Here's a very simple form, which assume you have an authenticated wallet reference (since we're using the wallet key as a unique identifier in the dynamodb database). I've also designed the DynamoDB schema according to best practice, which is to use a strong, unique identifier as the hash key, and then use sort-keys for scoping attributes. This results in a so called "sparse database", which is what DDB is designed to be. For more info on this, see https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/best-practices.html for technical stuff, and if you want to dive deep into this, I can't recommend this book enough - https://www.dynamodbbook.com - which is worth every, single cent if one wants to use cloud effeciently.
